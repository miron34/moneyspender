# utils/

Чистые функции — без побочных эффектов, без зависимостей от React.

## Файлы

### `format.ts` — форматтеры строк

- `fmt(n)` → `"1 840 ₽"` (NBSP перед `₽`, `ru-RU` локаль).
- `fmtK(n)` → `"24.9к"` для donut-центра. Для `n < 1000` возвращает строку числа.
- `hhmm(d)` → `"14:30"`.
- `fmtLabel(d, ref?)` → `"Сегодня, 14:30"` / `"Вчера, ..."` / `"25 апр, ..."`. Используется в строках трат.
- `dayHeader(d, ref?)` → `"Сегодня"` / `"Вчера"` / `"25 апреля"`. Заголовки групп в Истории.
- `fmtDatePill(d)` → `"25 апр 14:30"`. Для пилюли даты в форме добавления.

Все функции принимают опциональный `ref` (текущая дата) — для тестирования и стабильности.

### `date.ts` — фильтрация и группировка по датам

- `byPeriod(expenses, period, ref?)` — фильтр трат за `'day' | 'week' | 'month' | 'quarter'` (1, 7, 30, 90 дней).
- `byPrevPeriod(expenses, period, ref?)` — предыдущий аналогичный период (для сравнения в Аналитике).
- `groupByDay(expenses, ref?)` — группировка по календарному дню. Возвращает `DayGroup[]` отсортированных по убыванию даты.
- `periodDays(period)` — мапа период → количество дней.

### `analytics.ts` — агрегации

- `catStats(expenses)` → `[{id, total}, ...]` отсортированные по убыванию суммы.
- `sumAmount(expenses)` → сумма всех `amount`.
- `topN(expenses, n)` → top-N трат по убыванию суммы (для блока Топ-3 в Аналитике).
- `comparePeriods(current, previous)` → `{current, previous, deltaPct, deltaAmount}` для карточки сравнения.

## Принципы

- Чистые функции. Не используют стор, не делают сетевых запросов.
- Без мутаций входных массивов — всегда возвращают новый.
- Локаль везде `ru-RU`. Не интернационализируем (вне MVP).
