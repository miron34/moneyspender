## ✅ Сделано (MVP полностью работающий)

### Каркас
- Expo SDK 54 + Expo Router v6 + TypeScript strict
- Zustand-стор с моковой фикстурой (31 трата) для разработки
- Дизайн-токены (цвета, типографика, отступы, радиусы)
- Глобальные типы домена (Expense, Category, Period)
- Утилиты: format, date, analytics
- iPhone-фрейм 390×844 для web-режима

### UI и экраны
- **Главная** — переключатель периода (День/Неделя/Месяц), общий тотал, чарт-карточка с переключателем bar/donut, последние траты с адаптивным числом по высоте экрана (3-6 строк через `onLayout`)
- **История** — sticky-фильтры (категория + сортировка), группировка по дням, анимированная скролл-граница, swipe-to-delete с undo
- **Аналитика** — переключатель Нед./Месяц/Кв., карточка сравнения периодов с дельтой, мини-карточки (среднее за день / за неделю), полный барчарт с процентами, Топ-3 трат
- **Профиль** — аватар, список категорий с инлайн-редактированием, кнопка добавления, версия

### Анимации и взаимодействия
- BarChart и DonutChart — staggered анимация (55ms × index, 650ms, cubic-bezier(0.16, 1, 0.3, 1))
- BottomSheet на Reanimated v4 — slide-up, drag-to-close, springBack, keyboard avoidance с native-кривой iOS
- DrumPicker (5 колонок: день/месяц/год/час/мин) с scroll-snap и градиентами fade
- SwipeableRow — swipe-to-delete с двумя порогами (частичный → кнопка, полный → instant)
- UndoToast — отмена удаления в течение 3.5с (deferred delete: DELETE в БД отправляется только после истечения таймера)
- Haptics — light/medium/success/warning, с no-op fallback на web

### Синхронизация
- Supabase интеграция: lib/supabase.ts (client) + lib/sync.ts (transport layer)
- Optimistic-first: локальный update мгновенный, push в БД в фоне
- Seed: при первом запуске пустой БД льются 10 дефолтных категорий + 31 mock-трата
- Mock-fallback: если Supabase не настроен или fetch упал — приложение работает на in-memory моках
- ErrorToast для ошибок синхронизации

### Платформы и деплой
- Web: deploy на Vercel (auto-deploy с GitHub push), https://moneyspender.vercel.app
- iOS: тестируется через Expo Go (LAN режим)
- TypeScript strict + ESLint expo-config — оба проходят без warning'ов

## 📌 Известные ограничения

- **RLS отключён** — anon-ключ даёт публичный доступ к БД. Для личного проекта без публикации допустимо. Перед публикацией → шаг E.
- **Optimistic без rollback** — если push в Supabase упал, локальное состояние не откатывается, показывается только Toast. При следующем `fetchAll` локальное может расходиться с БД. Допустимо для одного пользователя; будет улучшено вместе с retry-очередью.
- **Mock fallback может скрывать сетевые проблемы** — если Supabase недоступен, приложение тихо работает на моках с `syncError`. Это удобно для разработки, но не подсказывает пользователю в production что сеть отвалилась.
- **На очень маленьких экранах** (iPhone SE, iPhone 8) количество "Последних трат" может быть всего 3 — это by design (Главная не скроллится).
