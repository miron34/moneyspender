## Главные находки

### 1. Модель «инжектирует» today, когда даты в фразе нет
Системный промпт прямо требует `date=null`, если дата не упомянута
(правило 1 в блоке «ПРАВИЛА ДЛЯ ДАТЫ»). На практике модель в ~30%
случаев без явного триггера времени всё равно ставит сегодняшнюю дату.

Проявления: B6 («две тысячи на продукты» → 2026-05-05),
B9 («1.5к на инет» → 2026-05-05), D18, D19.

Это не критично для UX (юзер увидит дату и скорректирует), но
формально нарушает контракт. Стоит подумать про:
- усиление правила в промпте (повторить запрет дважды + few-shot
  с явно фразой без даты → `date=null`),
- или серверный пост-процессинг: если в исходном тексте нет
  относительных триггеров (`вчера`, `сегодня`, `N назад`, дни недели,
  числовые даты) — снести `date` в null принудительно.

### 2. Сложные относительные даты ломаются
- C13 («неделю назад») → null вместо `today−7`. Модель не разворачивает
  «неделю» в 7 дней.
- C14 («в понедельник») → 2026-05-02 (суббота) вместо 2026-05-04.
  Модель путает день недели.
- C15 («25 декабря») → null вместо 2025-12-25. Не применяется правило
  «если число этого месяца > today, откатиться на год назад» —
  скорее всего модель отдаёт 2026-12-25 (будущее), а validate режет.

Гипотеза: few-shot покрывает «вчера», «позавчера», «X дней назад»,
«25 апреля», но не покрывает прямо «неделю назад», «в понедельник»,
«25 декабря» (когда дата ещё не наступила в текущем году). Стоит
добавить 3 примера в `prompt.ts`:
- `неделю назад X` → today−7
- `в понедельник X` (когда today=Tue) → today−1
- `25 декабря X` (когда today=май) → 2025-12-25

### 3. Директива комментария теряет регистр
E3: «напиши в комментарии пятёрочка» → name=«Пятёрочка», а должно быть
«пятёрочка» дословно. Промпт явно говорит «не редактируй, не сокращай».
Модель применяет общее правило капитализации имён собственных, не
понимая, что директива комментария отключает это.

Решение — либо ужесточить пример 9 в промпте (показать комментарий
с lowercase словом + комментировать «обрати внимание: имя сохранено
в исходном регистре»), либо постпроцессинг (но ему сложно отличить
«хотел капитализации» от «хотел дословно»).

реальный запрос идёт с `today=2026-05-05` — модель «прилипла» к числу
из примера. Лечится либо переписыванием примера на другую фразу/today,
либо принимается как допустимый шум LLM (±1 день для «недели» не ломает
UX).

### 4. Что точно работает хорошо
- Все базовые числительные («полторы тысячи», «3к», «сотка», «1.5к»).
- Сленг: «инет», «шава», «тачка», «пятёрочка» — все распознаются.
- **Динамические категории** (Group F): переименование label не сбивает,
  удалённые дефолтные не подставляются, кастомные с description ловятся,
  кастомные без description — ловятся по label.
- Директива «в категорию X» (E1, E2): и точная, и best-effort fallback.
- Серверная валидация (Group H): все 4 кейса возвращают корректные коды
  и `error`-строки.
